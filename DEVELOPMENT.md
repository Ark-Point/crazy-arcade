# 개발 과정 — 지시 이력으로 보는 프로젝트 히스토리

이 문서는 사용자가 Claude Code에게 어떤 순서로 지시했고, 각 지시가 어떻게 구현으로
이어졌는지를 기록한 것이다. 전 과정은 2026-06-12 하루 동안 대화형으로 진행되었다.

## 1. "크레이지 아케이드 게임 만들어줘. 웹 기반으로 멀티플레이 가능하게"

최초 한 줄 지시로 게임의 뼈대가 만들어졌다.

- **스택 결정**: Node.js + Express + Socket.IO (서버 권위적 30Hz 시뮬레이션) + HTML5 Canvas 클라이언트
- **핵심 메커니즘**: 물풍선 설치 → 3초 후 십자 물줄기 폭발(연쇄 폭발) → 맞으면 물방울에 갇힘(6초)
  → 시간 초과/적 터치 시 탈락 → 최후 1인 승리
- 닉네임 → 로비(방 목록/생성) → 대기실 → 게임 흐름, 박스 파괴 시 아이템 드롭(풍선/물줄기/스피드)
- 검증: 소켓 클라이언트 2개로 방 생성→입장→게임→폭발까지 도는 스모크 테스트 작성

## 2. "멀티 플레이는 어케 함" / "클라우드플레어 터널 띄워줘"

- 같은 PC(탭 여러 개), 같은 와이파이(로컬 IP), 인터넷(터널/배포) 세 가지 방법 안내
- `cloudflared` 설치 후 quick tunnel로 공개 URL 발급 → 외부 친구와 바로 플레이 가능하게 함

## 3. "움직임이 버벅이고 아이템도 없고 디자인도 구린데 전부 딥리서치해서 개선해줘"
##    (+ "애니메이션도 좀 더 이뻤으면 좋겠음")

딥리서치 워크플로우(병렬 에이전트 104개, 소스 22개, 주장 25개 교차 검증)를 돌려
세 가지를 근거 기반으로 개선했다.

- **넷코드**: 버벅임 원인이 "예측 없는 클라이언트"로 확정(입력 반영에 RTT 전체 소요).
  Valve/Gambetta 정석대로 **클라이언트 사이드 예측 + 서버 reconciliation**(서버·클라이언트가
  `public/shared.js`의 동일한 이동 코드 실행, 입력 시퀀스 번호, 미확인 입력 재적용) +
  **원격 플레이어 엔티티 보간**(~100ms 과거 렌더) 구현
- **아이템**: 원작 BnB 체계 반영 — 바늘(Ctrl/X로 갇힘 탈출), 신발(물풍선 밀기/킥),
  울트라(물줄기 최대), 롤러(속도)
- **비주얼/애니메이션**: 정적 타일 레이어 오프스크린 프리렌더, 캐릭터 걷기 바운스 +
  squash & stretch, 물풍선 펄스, 셀 연결형 물줄기, 파티클(파편/물방울/스파클), 화면 흔들림,
  카운트다운/GO! 연출

## 4. "게임 플레이 안내가 게임 화면 옆에 자세히 떴으면 좋겠음. 그리고 팀전 기능도 추가"

- 게임 화면 옆 상세 가이드 패널(조작법/규칙/아이템/승리 조건)
- 팀전: 대기실에서 모드 선택, 레드/블루 팀 배정, **갇힌 아군 터치 시 구출**,
  같은 편 진영 스폰, 한 팀 전멸 시 패배

## 5. "캐릭터 에셋 실제 에셋 웹검색해서 가져와서 캐릭터 선택도 할 수 있게"

- 실제 넥슨 에셋은 저작권 문제로 사용하지 않기로 하고(공개 서비스 중이었음),
  대신 **오리지널 캐릭터 6종**(버니/냥이/개구리/곰돌이/펭구/로보)을 Canvas 코드로 직접 디자인
- 대기실에 미리보기 썸네일 픽커, 선택 실시간 동기화, 팀전에서는 몸 색만 팀 색으로 변경

## 6. (원작 인게임 스크린샷 첨부) "이런 인게임 인터페이스를 만들어줘"

원작 레이아웃 구성만 차용해 오리지널 스타일로 재현:

- 우측 사이드바: 디지털 **TIMER**(2분 라운드 제한 신설), 플레이어 상태 카드(초상화+실시간 스탯),
  ITEM 박스, 게임 방법/나가기 버튼
- 하단 "내 아이템" 바(보유 현황 슬롯), 기존 가이드는 토글 오버레이로 이동

## 7. 게임플레이 피드백 4건 + "보스전도 딥리서치해서 만들어줘"

1. **물풍선 관통 이슈** → "겹침 감소만 허용(exit-only)" 충돌 규칙: 설치 직후 빠져나갈 수만 있고
   재진입/관통은 전원 차단 (회귀 테스트 포함)
2. **바늘이 너무 많음** → 드롭 테이블 정규화: 스탯템 82% / 유틸템 18%(바늘 14%→6%)
3. **기본 속도 답답** → 90px/s → 120px/s(초당 3칸), 롤러 증가폭 상향
4. **보스전**: 딥리서치(원작 몬스터 모드 + 보스 설계 모범 사례) 후 협동 PvE 구현 —
   보스 "킹 옥토"(3x3), 물줄기로 피격, **착탄 예고 장판(텔레그래프)** 물대포,
   착탄 후 **그로기 윈도우(2배 데미지)**, HP 66%/33% 페이즈 전환(분노/폭주, 쫄몹 스폰),
   접촉 시 갇힘 → 동료 구출, 처치 시 루찌 비 / 전멸·시간 초과 시 패배

## 8. "4대4까지 되게" + "이겼을 때 신나는 UI/UX"

- 방 정원 8인, 스폰 8지점(모서리+변 중앙), 팀당 4명, UI 8인 대응
- 승리 연출: 승자 전용 골드 **VICTORY!** 카드(트로피 바운스, 그라데이션 타이틀),
  폭죽+컨페티 5초, 우승 캐릭터 승리 댄스, 패배/무승부는 차분한 결과 카드

## 작업 방식 요약

- **딥리서치 2회**: 사실 검증된 근거(원작 메커니즘, 넷코드 이론, 보스 설계론) 위에서 구현.
  수치가 미기록인 부분(드롭률, 보스 HP 등)은 자체 밸런싱임을 명시
- **테스트 자동화**: 소켓 레벨 테스트 5종(스모크/팀전/풍선 관통/보스 루프/8인) +
  Playwright E2E 2종(실제 브라우저 플레이 + 스크린샷 검증)을 매 변경마다 실행
- **저작권 원칙**: 넥슨 에셋/아트는 복제하지 않고 메커니즘·레이아웃 개념만 참고,
  모든 그래픽은 Canvas 코드로 그린 오리지널
- **배포**: 로컬 서버 + Cloudflare quick tunnel로 즉시 외부 공유
